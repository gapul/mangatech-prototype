# フロントエンド担当 指示書

## 目的
- `prototype.html` を完成させる。
- `design.md` と `prototype.md` に完全準拠する。
- `image.png` のコメントUIを踏襲する。
- `001bj.pdf` のコマを使う。

## 参照ファイル
- `design.md`
- `prototype.md`
- `sns.md`
- `image.png`
- `001bj.pdf`

## 成果物
- `prototype.html`
- 画像アセット一式を `assets/panels/` に保存

## 画面要件
1. 長押しでコメント一覧が開く読書画面
2. コメント一覧シート（`image.png` 準拠）
3. コメント投稿画面（テキスト + 画像/動画追加 + タグ）
4. 複数コマ選択画面（近くのコマ/別巻の選択）
5. 自分の投稿一覧（プロフィール起点）
6. 他人のプロフィール + 投稿一覧

## UI仕様
1. カラーは `design.md` 指定の5色のみを主軸にする。
2. ダークUIを基本にし、カードやシートは白ベースで対比をつける。
3. 主要CTAは `#F20505`、ハイライトは `#F2CB05`。
4. フォントはゴシック系で太さのコントラストを作る。
5. 漫画的な演出として「集中線」「吹き出し風」「ステッカー風バッジ」を一部採用する。

## コメント一覧UIの再現
1. 上部に「× / コメント / …」のヘッダーを置く。
2. 「いいジャン順 / 新着順」のタブを表示する。
3. 各コメントは「アイコン / 名前 / 本文 / 日時 / いいジャン数」を1行ブロックで並べる。
4. 最下部に赤いCTAボタンを固定配置する。

## コマ画像の取り出し
1. `001bj.pdf` から4〜6枚のコマを切り出す。
2. 切り出した画像は `assets/panels/` に配置する。
3. 解像度は幅700〜1200pxを目安に統一する。
4. コマの種類は「街の俯瞰」「食事」「表情アップ」「驚き表情」などバリエーションを持たせる。

## 受け入れ基準
1. `prototype.md` に書かれた全機能が画面で表現されている。
2. `image.png` と同じ構造でコメントUIが再現されている。
3. `design.md` のトーンから逸脱していない。
4. どの画面も「コマ画像」が主役として見える。

## チェック手順
1. `prototype.html` をローカルで開き、6画面が揃っていることを確認する。
2. コメントUIが `image.png` と同じ構造になっていることを確認する。
3. 主要CTAが必ず赤色で目立つことを確認する。

## デプロイ方針（Option A）
1. `prototype.html` と `assets/` を GitHub Pages で静的公開する。
2. 公開ディレクトリはリポジトリ直下とし、`prototype.html` を入口にする。
3. 画像が重い場合は事前に圧縮して配置する。
